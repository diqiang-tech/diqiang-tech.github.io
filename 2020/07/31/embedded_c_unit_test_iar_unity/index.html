浅尝IAR下为嵌入式C构建测试驱动开发

发表于 2020-07-31 更新于 2019-09-06 分类于 TDD 阅读次数：
本文字数： 2.7k 阅读时长 ≈ 2 分钟

我们以往的开发模式，往往没有基于测试驱动开发（Test Driven Development,TDD）。测试这块以功能测试，系统集成测试为主，集成测试中也会自己设计一些自动化测试的工装，真正意义上的基于代码级的单元测试是空白。

测试驱动开发（Test Driven Development），简称TDD。我们不要把它从字面上理解成是某种驱动的开发，或者这个应该是由测试工程师去关注的某种技术。实际上这应该是开发人员去关注的一种开发模式，一种基于测试驱动去开发的模式，它是敏捷开发的一项核心技术。

以下我们以IAR下的C开发为例，聊一聊如何进行具体的单元测试工作。

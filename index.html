浅尝IAR下为嵌入式C构建测试驱动开发

发表于 2019-08-20 分类于 TDD
本文字数： 2.7k 阅读时长 ≈ 2 分钟

我们以往的开发模式，往往没有基于测试驱动开发（Test Driven Development,TDD）。测试这块以功能测试，系统集成测试为主，集成测试中也会自己设计一些自动化测试的工装，真正意义上的基于代码级的单元测试是空白。

测试驱动开发（Test Driven Development），简称TDD。我们不要把它从字面上理解成是某种驱动的开发，或者这个应该是由测试工程师去关注的某种技术。实际上这应该是开发人员去关注的一种开发模式，一种基于测试驱动去开发的模式，它是敏捷开发的一项核心技术。

以下我们以IAR下的C开发为例，聊一聊如何进行具体的单元测试工作。

Hexo+Next+Github搭建个人博客

发表于 2019-07-14 更新于 2019-07-24 分类于 hexo
本文字数： 3.3k 阅读时长 ≈ 3 分钟

本地环境搭建

本文作者基于windws 10系统下进行本地环境的搭建。

安装git

点此选择相应的版本下载并安装。

安装Node.js

点此选择相应的版本下载并安装。

安装确认

找到gitbash，点击进入控制台窗口。

$ git version #查看git版本信息

$ node -v #查看Node.js版本信息

$ npm -v #查看包管理工具版本信息

安装hexo

npm install -g hexo-cli

初始化

在gitbash下，切换到你准备存放博客的目录

$ hexo init my-blog

执行以上操作将会花费一些时间，完成后将会在所在的目录下创建my-blog文件夹，所有本地的文件都会存放在这个文件夹下面。

然后执行以下操作：

$ cd my-blog

$ npm install

本地预览

在完成了本地初始化后，如果没有任何报错，那么可以在gitbash控制台窗口中执行

$ hexo g

$ hexo s

之后显示如下信息：

INFO Start processing
INFO Hexo is running at http://localhost:4000 . Press Ctrl+C to stop.

在网络浏览器的地址栏中输入http://localhost:4000

部署到github

注册github帐号

如果没有github帐号，先注册帐号。记下这里的用户名，假设你在注册github帐号时的用户名是username.

创建一个仓库，仓库名为：username.github.io，在该仓库创建一个“index.html”,随意写一些内容进去。这时候在浏览器中就可以用username.github.io就可以访问了。

配置SSH key

要把本地的内容通过git部署到github的话需要配置以下SSH key。

执行以下命令，配置一下本地账户：

$ git config –global user.name “username”

$ git config –global user.email “xxx@xxx.xxx“

生成密钥：

$ ssh-keygen -t rsa -C “xxx@xxx.xxx“

按照提示敲三次回车键就可以生成密钥文件

可以执行以下命令把密钥拷贝到剪贴板中：

$ clip < ~/.ssh/id_rsa.pub

添加到本地可信列表：

$ ssh -T git@github.com

如果返回Hi xxx! You’ve successfully authenticated, but GitHub does not provide shell access.
表明添加成功。

在github中添加SSH key：

到github中添加一个SSH key，标题随意填。key就是上面生成的。

部署

修改_config.yml

deploy:

type: git

repo: git@github.com:username/username.github.io.git

branch: master

安装部署插件

$ npm install hexo-deployer-git –save

上传

$ hexo clean

$ hexo g

$ hexo d

浏览

在网络浏览器中输入https://username.github.io就可以浏览你刚刚部署到github上的博客了。

安装Next主题

  • 在gitbash中将目录切换到my-blog
  • 执行 git clone https://github.com/theme-next/hexo-theme-next themes/next
  • 目录切换到my-blog/themes/next
  • 执行 git pull
